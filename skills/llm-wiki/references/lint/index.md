# wiki-lint — 健康审计 + 图谱维护统一入口

> **链接格式**：所有内部链接使用思源原生块引用 `((<doc-id> "display text"))`。思源不解析 `[[wikilink]]` markdown。详见 `references/constitution.md`。
>
> 前置步骤和共享操作定义见 `shared.md`；存储宪法见 `references/constitution.md`。

此技能是 wiki 健康和图谱维护的统一路由入口。根据用户意图分发到 9 种模式之一。

## 模式路由

从用户措辞推断模式，加载对应 reference 后执行：

| 用户意图 / 关键词 | 模式 | 文件 | 风险 |
|---|---|---|---|
| "审计 wiki"、"什么需要修复"、"wiki 健康检查"、未指定 | **审计** | `audit.md` | 只读 |
| "修复"、"自动修复"、"consolidate" | **Consolidate** | `consolidate.md` | 写入 |
| "日常更新"、"定期维护" | **日常更新** | `daily.md` | 写入 |
| "状态如何"、"wiki 状态" | **状态审计** | `daily.md` | 只读 |
| "去重"、"查找重复页面"、"合并重复" | **去重** | `dedup.md` | **破坏性** |
| "交叉链接"、"添加链接"、"连接页面" | **交叉链接** | `crosslink.md` | 写入 |
| "标签规范化"、"标签审计"、"整理标签" | **标签分类法** | `taxonomy.md` | 写入 |
| "给图谱上色"、"颜色"、"可视化" | **图谱着色** | `colorize.md` | 写入 |
| "洞察"、"枢纽"、"中心"、"wiki 结构" | **洞察分析** | `insights.md` | 只读 |

**默认**：未指定模式时 → 审计模式（只读，安全）。

## 执行流程

```
用户请求 → 关键词路由 → 加载对应 reference
                ↓
         执行 shared.md 前置步骤（配置 + 权限 + 注册表）
                ↓
         运行对应模式的具体流程
                ↓
         按 shared.md 更新跟踪文件 + 写入日志
```

## 文件索引

| 文件 | 内容 | 权限 |
|---|---|---|

| `dedup.md` | 去重 — 身份解析 + 页面级合并（含重定向存根） | **破坏性** |
| `crosslink.md` | 交叉链接 — 自动化块引用 + 关系类型推断 | 写入 |
