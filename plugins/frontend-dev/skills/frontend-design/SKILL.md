本 skill 指导你做出有辨识度、可上生产的前端界面，摆脱千篇一律的"AI 流水线"审美。写真正能跑的代码，在审美细节与创意取舍上格外用心。

用户会给出前端需求：要做的组件、页面、应用或界面，可能附带用途、受众或技术约束。

## 设计思考

动手写代码前，先吃透场景，定下一个**大胆**的审美方向：
- **用途**：这个界面解决什么问题？谁来用？
- **调性**：选一个极致的风格——极简到狠、繁复到乱、复古未来、自然有机、奢华精致、玩具般俏皮、杂志编辑风、粗野原始、装饰艺术几何、柔和马卡龙、工业实用，等等。风味多得很。拿这些当灵感，但要设计出一个忠于既定方向的调性。
- **约束**：技术要求（框架、性能、无障碍）。
- **差异化**：什么让它**令人难忘**？哪一处是别人会记住的？

**要紧**：选定一个清晰的概念方向，再精准执行。大胆的繁复与克制的极简都行——关键在用心，不在用力。

然后写出能跑的代码（HTML/CSS/JS、React、Vue 等），要做到：
- 可上生产，功能完整
- 视觉抢眼，令人过目不忘
- 风格统一，审美主张鲜明
- 每个细节都打磨到位

## 前端审美准则

重点抓这几样：
- **字体**：选漂亮、独特、有意思的字体。避开 Arial、Inter 这类大路货，挑能抬升整体审美的——出人意料、有个性的字体。用一款有辨识度的标题字，配一款考究的正文字。
- **色彩与主题**：认定一套统一的审美。用 CSS 变量保持一致。一个主色配上利落的点缀色，胜过畏首畏尾、平均分布的调色板。
- **动效**：用动画做效果和微交互。HTML 优先用纯 CSS 方案，React 有 Motion 库就用。把力气集中在高光时刻：一次编排得当的页面载入，配上错落渐显（animation-delay），比一堆零散的微交互更讨喜。善用滚动触发，和出人意料的悬停态。
- **空间构图**：布局出人意料。非对称、重叠、对角线走向、打破网格的元素。要么大方留白，要么收着堆密度。
- **背景与视觉细节**：营造氛围和层次，别一上来就用纯色。加上契合整体审美的效果和纹理。试试渐变网格、噪点纹理、几何图案、分层透明、戏剧化阴影、装饰边框、自定义光标、颗粒叠加这些花样。

**绝不**套用千篇一律的 AI 审美：用滥的字体（Inter、Roboto、Arial、系统字体）、老套的配色（尤其白底紫渐变）、可预料的布局和组件套路、缺乏场景个性的模板化设计。

放开手脚去理解需求，做出出人意料、真正为这个场景而生的选择。没有两份设计该长得一样。明暗主题、字体、审美都换着来。**绝不**在多次生成里都收敛到同一个常见选择（比如 Space Grotesk）。

**重要**：实现的复杂度要配得上审美设想。繁复风需要精雕细琢的代码，配大量动画与效果；极简或考究风则要克制、精准，在间距、字体和微妙细节上下功夫。优雅来自把设想执行到位。

记住：Claude 能做出非凡的创意。别保留——跳出框框、全心投入一个独特的设想，把真正能造出的东西亮出来。

## 参照现成的品牌设计系统

要照某个知名产品的风格来做，或者想要一个具体、可模仿的方向时，先翻 `references/INDEX.md` 挑一个品牌，再读 `references/design-md/<brand>/DESIGN.md`，把里面的配色、字体、组件 token 当作设计依据。没有合适的品牌，就回到上面的方法论，自己定方向。
