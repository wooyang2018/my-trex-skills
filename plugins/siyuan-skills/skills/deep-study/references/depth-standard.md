# Depth Standard — L1-L4 深度标准与面试追问链

定义 deep-study 的深度判定标准，使"深度"可检验而非主观。每个 concept 页面必须达到 L4 深度。

## L1-L4 深度层级

| 层级 | 判定标准 | 达标标志 | 写入位置 |
|---|---|---|---|
| L1 定义 | 能用一句话说清"X 是什么" + 列出 2-3 个核心特征 | Flashcards L1 填空题能独立回想 | `## Flashcards` L1 段（`custom-card-type=cloze`）+ 定义段 |
| L2 原理 | 能讲清"X 怎么工作"的机制 / 流程，不含糊 | Flashcards L2 问答题能答出关键步骤 | `## Flashcards` L2 段（`custom-card-type=qa`）+ `## How It Works` |
| L3 动机 | 能回答"为什么这样设计" + 列出被放弃的替代方案及权衡 | Flashcards L3 单选题能选对并理解选项差异 | `## Flashcards` L3 段（`custom-card-type=single-choice`）+ `## Why This Way` |
| L4 实战 | 能应对面试官追问 + 知道生产踩坑 + 有实践方案 | Flashcards L4 多选题 + Interview Questions + Common Pitfalls + Practice Ideas 预填 | `## Flashcards` L4 段（`custom-card-type=multi-choice`）+ 深入段 |

## L4 达标要求（deep-study 独有）

标准 ingest 把深入段留空给人工。deep-study 必须预填能从研究素材推断的实战内容：

### Interview Questions（面试追问）

预填 3-5 个面试官可能追问的问题 + 答题方向（切入思路，非完整答案）。问题须从 L1 递进到 L4：

- L1 级追问："X 是什么？"（基础确认）
- L2 级追问："X 内部是怎么实现的？"（机制深挖）
- L3 级追问："为什么用 X 而不用 Y？权衡是什么？"（对比动机）
- L4 级追问："生产环境踩过什么坑？怎么调优？"（实战）

### Common Pitfalls（项目踩坑）

预填 2-3 个常见误区 / 陷阱 / 踩坑，每个含：现象、原因、解法。从研究素材（博客 / issue / SO）提取真实案例，不杜撰。无法从素材推断的留空并标注待深化。

### Practice Ideas（实践方案）

预填 1-2 个练手方向：小项目 / 代码示例 / 架构设计。须可落地，不是空泛建议。

## 面试追问链设计方法

追问链是 deep-study 的核心产出之一（见 `deliverable.md`），设计方法：

1. **从 L1 起步**：用最基础的定义问题开场，确认对方懂概念。
2. **逐层递进**：L1→L2→L3→L4，每层 1-3 个问题，后一层建立在前一层答案上。
3. **答题方向非完整答案**：每个问题附"答题方向"——答题的切入思路、关键点、易踩的坑，不写完整答案（完整答案在 concept 正文）。
4. **L4 聚焦实战**：最后一层必须是生产环境问题（踩坑 / 调优 / 选型），这是区分"背了概念"和"真正做过"的关键。
5. **5-10 个问题**：太少覆盖不了深度，太多冗余。典型 6-8 个。

## 示例追问链（主题：Go 垃圾收集）

| # | 层级 | 问题 | 答题方向 |
|---|---|---|---|
| 1 | L1 | Go 的 GC 用的是什么算法？ | 三色标记 + 并发标记清除，非分代非压缩 |
| 2 | L2 | 三色标记具体怎么工作？白 / 灰 / 黑的含义和流转？ | 白=未访问、灰=已访问但子节点未扫、黑=完成；从灰队列出发扫子节点 |
| 3 | L2 | 并发标记怎么处理 GC 期间对象引用变化？ | 混合写屏障（Dijkstra+Yuasa），黑对象引用白对象时标灰 |
| 4 | L3 | 为什么 Go 不用分代 GC？ | Go 栈上分配 + 逃逸分析减少堆压力；分代收益小且增加复杂度；权衡 simplicity |
| 5 | L3 | 写屏障为什么从 Dijkstra 切到混合写屏障？ | 1.8 前栈重扫描成本高；混合写屏障消除栈重扫描，降低 STW |
| 6 | L4 | 生产中遇到 STW 尖刺怎么排查？ | GODEBUG=gctrace=1 看 GC 频率；pprof 找分配热点；GOGC 调阈值或用 GOMEMLIMIT |
| 7 | L4 | 怎么减少 GC 压力？ | 减少堆分配（sync.Pool / 复用 / 值类型）；预分配 slice；避免大对象 |

这个追问链从定义到生产调优，覆盖 L1-L4。每个答题方向是切入思路，完整答案在 concept 页面正文。

## 深度不达标的处理

研究素材不足以支撑 L4 时（如新主题缺乏实战案例）：

1. 仍创建 concept 页面，L1-L4 闪卡正常注册。
2. 深入段预填能推断的内容，无法推断的留空并加 `> [!CAUTION] 此段待深化，研究素材不足`。
3. `custom-status=draft`，日志注明"L4 待深化：<具体缺口>"。
4. 在 `## Practice Ideas` 写"建议通过 <具体实践> 补充实战经验"。

不伪造实战内容——L4 的价值在于真实性，杜撰的踩坑比留空更有害。
